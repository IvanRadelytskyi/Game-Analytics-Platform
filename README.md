# Game Analytics Platform 🚀

[📊 Переглянути інтерактивний дашборд на Tableau Public](https://public.tableau.com/views/GameAnalyticsPlatform/Executive?:language=en-US&:sid=&:redirect=auth&:display_count=n&:origin=viz_share_link)

Комплексний енд-ту-енд (End-to-End) проєкт із продуктової та фінансової аналітики ігрової платформи. У цьому кейсі реалізовано повний цикл роботи аналітика: від написання складних SQL-запитів для трансформації сирих транзакційних даних до побудови BI-архітектури та розрахунку ключових бізнес-метрик (MRR, Churn, LTV, ARPPU, Lifetime).
---
## 🛠 Стек технологій
* **Бази даних:** SQL (PostgreSQL) — очищення, агрегація, віконні функції, розрахунок когорт
* **BI-візуалізація:** Tableau Desktop — проектування UX/UI дашбордів, розрахункові поля
* **Продуктові метрики:** Cohort Retention, Churn Rate, LTV, ARPPU, Lifetime (LT)

## 📊 Фінальний аналітичний звіт

### 1. Верхньорівневий стан бізнесу (KPI)
Загальний фінансовий зріз платформи за звітний період показує солідний об'єм регулярного виторгу, але підсвічує критичну вразливість продукту:
* **Загальний MRR (Monthly Recurring Revenue):** **$63,14K**.
* **База платників:** **1 392** унікальних активних користувачів.
* **Середній Churn Rate:** **38,43%**.

> ⚠️ **Бізнес-контекст:** Для ігрової індустрії щомісячний відтік на рівні ~38% — це критично високий показник (норма становить 5-15%). Компанія працює в режимі «дірявого відра»: вона змушена постійно спалювати бюджети на агресивне залучення (UA), щоб просто перекривати масовий відтік поточної бази платників.

### 2. Крос-аналіз січневої "аномалії" та технічний інсайт
На фінальних графіках чітко простежується катастрофічне падіння метрик у січні: рекордний поведінковий відтік (**-189 юзерів**), фінансові втрати (**-$8 440 Churned Revenue**) та падіння Lifetime до позначки **1,000**.

🔧 **Головний технічний інсайт:** 
Аналіз вихідної бази даних показав, що реальних транзакцій за січень у датасеті немає — база обривається на грудні 2022 року. Січень з'явився на графіках як *розрахунковий фантомний місяць* через логіку функцій порівняння періодів (зміщення дат / `LOOKUP`). BI-платформа інтерпретувала раптове закінчення даних як 100% відтік усієї активної бази користувачів у наступному періоді. Це наочний кейс того, як математичні моделі реагують на технічний обрив часових рядів.

### 3. Сегментація аудиторії: Вік та Метрики LTV / ARPPU
Аналіз вікових когорт та ліній трендів (Scatter Plot) чітко розділяє аудиторію на драйверів та аутсайдерів:
* **Когорта "18-24" (Молодь) — Ядро монетизації:** Має найвищий показник **LTV ($49,50)** та максимальний середній чек за місяць (**ARPPU $50,34**). Лінія тренду для **Game 3** підтверджує, що гра є абсолютним молодіжним хітом — чим молодший гравець, тим охочіше він донатить.
* **Когорта "35-44" — Пасивний сегмент:** Демонструє найнижчі показники, де LTV падає до **$27,78**. 

### 4. Мовний розріз (User Language)
Сегментація показала повне домінування локального ринку: **українська та російська мови** генерують понад 95% усього платоспроможного трафіку та доходу. Міжнародний стандарт (англійська мова) у цьому конкретному портфелі ігор є пасивним та не несе фінансової цінності.

## 💡 Стратегічні рекомендації для бізнесу

1. **Оптимізація UA (User Acquisition) для Game 3:** Повністю зупинити маркетингові кампанії для аудиторії 35+. Весь бюджет закупівлі трафіку сфокусувати на сегменті 18-24 роки, де окупність користувача майже вдвічі вища.
2. **Реінжиніринг системи Retention:** Оскільки продукт вміє залучати та монетизувати першу покупку, але Lifetime падає до 1 місяця, необхідно терміново впроваджувати внутрішньоігрові механіки утримання другого місяця (щоденні квести/нагороди, системи лояльності, сезонні абонементи Battle Pass).
3. **Релокація бюджетів підтримки:** Заморозити витрати на розширення англомовної локалізації. Переспрямувати ресурси на розвиток українського ком'юніті та локальної служби підтримки (Customer Support), які безпосередньо утримують платоспроможну базу платформи.
4. **Консолідація портфеля ігор:** Оскільки Game 1 та Game 2 мають мізерну кількість платників та фактично існують лише на папері, бізнесу варто розглянути варіант їх закриття або заморозки, щоб сфокусувати всі ресурси розробки на масштабуванні єдиного успішного продукту — Game 3.
